v7.0 / vorterm

Vorterm 7.0

Setup de Windows Terminal + PowerShell 7 + oh-my-posh + Nerd Font, en un GUI. Profile optimizado, cache OMP, lazy load de modulos. Esta guia: como usarlo, donde estan las cosas, como cambiar el tema.

01Que es

Un script con GUI que deja Windows Terminal listo en 1 click:

  • PowerShell 7 instalado y por defecto.
  • Nerd Font (JetBrainsMono) para iconos y glyphs en el prompt.
  • oh-my-posh con tema darkblood (cambiable, ver mas abajo).
  • Profile optimizado: cache de inicio, lazy load de posh-git + Terminal-Icons, aliases utiles, fallback prompt.
  • Settings.json de Windows Terminal parcheado (font correcto, comandline pwsh, dir inicial).

02Instalar

  1. Doble click en INSTALAR.bat (o ejecutar setup_terminal.ps1 en pwsh elevada).
  2. En la GUI: marcar componentes, pulsar Run.
  3. Cerrar todas las ventanas de Windows Terminal y reabrir.
reabrirTras instalar, cierra todas las ventanas WT. Una sola pestaña abierta puede mantener el font/theme viejos cargados.

cacheEl profile cachea la inicializacion OMP en omp-init-<hash>.ps1. El hash depende del path del tema, asi cambiar de tema en el profile genera un cache nuevo automaticamente. Si edits el tema activo, su mtime invalida el cache propio.

04Cambiar tema

  1. Descarga el .omp.json nuevo a %USERPROFILE%\.poshthemes\: 
    powershell$theme = 'kushal'  # nombre sin .omp.json
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/JanDeDobbeleer/oh-my-posh/main/themes/$theme.omp.json" `
                  -OutFile "$env:USERPROFILE\.poshthemes\$theme.omp.json"
  2. Edita el profile: notepad $PROFILE. Cambia darkblood.omp.json por el nombre del tema nuevo (2 sitios).
  3. Reabre todas las ventanas de Windows Terminal.
tipEl profile nuevo cachea por hash del path → cambiar tema regenera cache automatico. Si tienes profile viejo, borra $env:LOCALAPPDATA\Vorterm\omp-init.ps1 manualmente.

06Problemas comunes

WT muestra warning de fuente / iconos como cuadritos

"No se puede encontrar la fuente X. Se selecciono Y en su lugar."

El nombre de la fuente en settings.json no es el que DirectWrite resuelve. Fix:

powershell$wt = "$env:LOCALAPPDATA\Packages\Microsoft.WindowsTerminal_8wekyb3d8bbwe\LocalState\settings.json"
Copy-Item $wt "$wt.bak"
$j = Get-Content $wt -Raw | ConvertFrom-Json -Depth 100
$j.profiles.defaults.font.face = 'JetBrainsMono Nerd Font Mono'
foreach ($p in $j.profiles.list) { if ($p.font) { $p.font.face = 'JetBrainsMono Nerd Font Mono' } }
$j | ConvertTo-Json -Depth 100 | Set-Content $wt -Encoding utf8

Cierra todas las ventanas WT y reabre.

Cambie el tema en el profile pero no aplica

Profile dice tema X, pero se ve el viejo.

Cache OMP no invalido. Borra y reabre:

powershellRemove-Item "$env:LOCALAPPDATA\Vorterm\omp-init*.ps1" -Force

Glyphs como ?? o cuadritos

El prompt no muestra iconos.

Falta Nerd Font o WT no usa font Nerd. WT → Configuracion → tu perfil → Apariencia → Fuente: JetBrainsMono Nerd Font Mono.

"Comando no encontrado: oh-my-posh"

Profile carga pero no hay prompt custom.

Reinstalar via Vorterm o manual:

powershellwinget install JanDeDobbeleer.OhMyPosh -s winget

Profile roto, terminal no carga

Errores rojos al abrir terminal.
powershellMove-Item $PROFILE "$PROFILE.bak"
# Reabrir terminal — sin profile carga limpio. Re-ejecuta Vorterm para regenerar.

07Desinstalar

El setup_terminal.ps1 incluye GUI de reset. Tambien puedes:

  • Volver al profile original: Move-Item $PROFILE "$PROFILE.bak"
  • Restaurar settings.json WT: usar el backup automatico settings.json.bak.* que crea Vorterm.
  • Quitar oh-my-posh: winget uninstall JanDeDobbeleer.OhMyPosh
  • Quitar Nerd Font: Panel control → Fuentes → buscar JetBrainsMono → eliminar.
no tocaVorterm no desinstala PowerShell 7, Git, Windows Terminal ni modulos Az/Graph/EXO.
Para devs: internals del setup script y CSV drop-in

WPF + PowerShell, sin compilacion. XAML inline, runspaces para async, Synchronized hashtable para IPC, Dispatcher para mutar UI desde workers.

Detalles cache OMP

El profile cachea init en %LOCALAPPDATA%\Vorterm\omp-init-<hash>.ps1. Hash MD5(8) del path del tema → cada tema tiene cache propio. Invalidacion via mtime del tema vs cache.

Detalles fuente Nerd Font

JetBrainsMono Nerd Font v3 expone 2 nombres en tabla name del TTF:

  • name ID 1 (GDI family): JetBrainsMono NFM (lo que enumera InstalledFontCollection)
  • name ID 16 (typographic / DWrite): JetBrainsMono Nerd Font Mono (lo que WT espera)

Si settings.json tiene el GDI family, WT muestra warning. setup_terminal.ps1 mapea GDI → typographic antes de escribir.
